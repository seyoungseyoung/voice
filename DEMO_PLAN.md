# Sentinel-Voice 데모 계획서

## 목표
DACON '피싱·스캠 예방 서비스 개발 경진대회' MVP 시연 및 심사

## 데모 구성

### 1. 시연 환경
- **형식**: 웹 기반 데모 (브라우저에서 실행)
- **백엔드**: FastAPI 서버 (이미 완성)
- **프론트엔드**: HTML/CSS/JavaScript 단일 페이지
- **배포**: 로컬 또는 클라우드 (선택사항)

### 2. 핵심 기능 시연

#### A. 실시간 텍스트 분석
- 입력: 통화 내용 텍스트
- 출력:
  - 위험도 점수 (0-100)
  - 위험 레벨 (SAFE/LOW/MEDIUM/HIGH/CRITICAL)
  - 탐지된 피싱 기법
  - 개인정보 마스킹 결과
  - 유사 피싱 사례

#### B. 음성 파일 분석
- 입력: 녹음된 통화 파일 (.wav, .mp3)
- 처리: STT → 텍스트 분석
- 출력: 위와 동일 + 음성 전사(transcript)

#### C. 실시간 시각화
- 위험도 게이지 차트
- 구성 점수 비교 (Keyword/Sentiment/Similarity)
- 시간에 따른 위험도 변화 (선택사항)

## 데모 시나리오

### 시나리오 1: 검찰 사칭 보이스피싱
**통화 내용:**
> "안녕하세요, 서울중앙지검 김철수 검사입니다. 당신 명의의 계좌가 보이스피싱 범죄에 사용되었습니다. 지금 바로 확인하지 않으면 내일 체포영장이 발부됩니다. 주민등록번호와 계좌번호를 말씀해주세요."

**예상 결과:**
- 위험도: 85-95점 (HIGH/CRITICAL)
- 탐지 기법: 기관 사칭, 협박, 개인정보 요구
- 권장 조치: 즉시 통화 종료 및 112 신고

### 시나리오 2: 금융감독원 사칭
**통화 내용:**
> "금융감독원입니다. 고객님 계좌에서 이상거래가 감지되었습니다. 즉시 안전계좌로 자금을 이체하셔야 피해를 막을 수 있습니다. 카드번호와 OTP 번호를 알려주세요."

**예상 결과:**
- 위험도: 80-90점 (HIGH)
- 탐지 기법: 기관 사칭, 개인정보 요구, 금전 요구
- 권장 조치: 의심스러운 통화, 개인정보 제공 금지

### 시나리오 3: 정상 통화 (대조군)
**통화 내용:**
> "안녕하세요. 택배 배송 관련하여 연락드렸습니다. 오늘 오후 2시경 방문 예정인데 댁에 계실까요?"

**예상 결과:**
- 위험도: 10-20점 (SAFE)
- 탐지 기법: 없음
- 권장 조치: 정상 통화

## 기술적 차별화 포인트

### 1. 하이브리드 접근 방식
- **On-device**: 빠른 1차 필터링 (키워드, 감정 분석)
- **Cloud**: 정확한 2차 분석 (Vector DB 유사도 검색)

### 2. RAG 기술 활용
- 실제 피싱 사례 DB와 실시간 비교
- 새로운 피싱 패턴 학습 가능

### 3. Privacy-First 설계
- PII 자동 마스킹
- 민감 정보 서버 전송 전 처리
- GDPR/개인정보보호법 준수

### 4. 다층 Risk Scoring
- Keyword Score (30%): 피싱 관련 키워드
- Sentiment Score (30%): 공포/긴급성 감지
- Similarity Score (40%): 기존 사례와 유사도

## 성능 지표

### 목표 성능
- **정확도**: 90% 이상 (피싱 탐지율)
- **오탐률**: 10% 이하
- **응답 시간**:
  - 텍스트 분석: < 1초
  - 음성 분석: < 5초 (30초 통화 기준)
- **처리량**: 초당 10건 이상

### 평가 방법
1. 실제 피싱 사례 데이터셋 테스트
2. 정상 통화 데이터로 오탐률 측정
3. 속도 벤치마크
