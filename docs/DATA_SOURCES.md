# 데이터 소스 계획

## 데이터셋 구성

### 1. 정상 통화 데이터 (Normal Conversations)

#### AI Hub - 민원 질의응답 데이터
- **출처**: https://aihub.or.kr
- **데이터셋**: "민원(콜센터) 질의-응답 데이터"
- **내용**: 공공기관/기업 고객센터 통화 녹음
- **규모**: 약 10,000건 이상
- **용도**: 정상 통화 학습, 오탐률(False Positive) 감소
- **라벨**: `[정상]`

**특징:**
- 실제 고객 상담 대화
- 다양한 주제 (민원, 문의, 불만 등)
- 음성 품질 우수
- 전사(Transcript) 포함

#### AI Hub - 한국어 대화 음성 데이터
- **출처**: https://aihub.or.kr
- **데이터셋**: "한국어 대화 음성"
- **내용**: 일상 대화, 전화 통화
- **용도**: 정상 통화 패턴 학습

### 2. 보이스피싱 데이터 (Phishing Data)

#### 금융감독원 - 그놈 목소리
- **출처**: https://voice.fss.or.kr
- **내용**: 실제 보이스피싱 녹음 파일 (재연)
- **규모**: 약 100건
- **용도**: 피싱 패턴 학습
- **라벨**: `[피싱]` + `[검찰사칭]`, `[금감원사칭]` 등

**수집 방법:**
1. 웹사이트 방문
2. 각 사례별 음성 재생
3. 브라우저 개발자 도구로 오디오 URL 확인
4. 다운로드 (저작권 확인 필요)

#### 경찰청/금감원 보도자료
- **출처**:
  - 경찰청: https://www.police.go.kr
  - 금융감독원: https://www.fss.or.kr
- **내용**: 보이스피싱 사례 텍스트
- **용도**: 피싱 스크립트 패턴 분석

#### YouTube 공개 자료
- **출처**: YouTube 검색 "보이스피싱 실제 사례"
- **내용**:
  - 뉴스 보도 중 재연 음성
  - 경찰/금감원 예방 교육 영상
  - 피해자 증언 (음성 변조)
- **주의**: 저작권 확인 필수

#### 자체 제작 데이터
- **방법**: 시나리오 기반 음성 녹음
- **시나리오**:
  1. 검찰/경찰 사칭
  2. 금융감독원 사칭
  3. 은행/카드사 사칭
  4. 국세청 사칭
  5. 택배/우체국 사칭
- **규모**: 50-100건 목표
- **라벨**: 상세 태깅 ([접근], [협박], [개인정보요구], [송금유도])

### 3. 텍스트 기반 데이터 (보완용)

#### 온라인 커뮤니티 사례
- **출처**:
  - 보배드림 자유게시판
  - 인스티즈
  - 네이버 카페 (보이스피싱 피해자 모임)
- **내용**: 실제 피해 사례 텍스트
- **용도**: 키워드 패턴 분석

#### 공공데이터 포털
- **출처**: https://www.data.go.kr
- **검색어**: "보이스피싱", "금융사기", "전화금융사기"
- **내용**: 통계 데이터, 신고 내역

## 데이터 통합 전략

### 학습 데이터셋 구성
```
총 데이터: 10,000 - 15,000건
├─ 정상 통화: 8,000건 (80%)
│  ├─ AI Hub 민원 데이터: 7,000건
│  └─ AI Hub 일상 대화: 1,000건
└─ 보이스피싱: 2,000건 (20%)
   ├─ 금융감독원: 100건
   ├─ 자체 제작: 100건
   ├─ 텍스트 변환: 1,500건
   └─ 증강 데이터: 300건
```

### 데이터 증강 (Data Augmentation)
1. **텍스트 변형**
   - 동의어 치환
   - 문장 순서 변경
   - 노이즈 추가

2. **음성 변형**
   - 속도 조절 (0.9x ~ 1.1x)
   - 피치 변경
   - 배경 소음 추가
   - 음성 변조

3. **시나리오 조합**
   - 기관명 교체 (검찰 ↔ 경찰 ↔ 금감원)
   - 협박 문구 조합
   - 개인정보 종류 변경

## 라벨링 체계

### 1차 라벨 (Binary)
- `NORMAL`: 정상 통화
- `PHISHING`: 보이스피싱

### 2차 라벨 (Phishing Type)
- `검찰/경찰 사칭`
- `금융기관 사칭`
- `국세청 사칭`
- `택배/우체국 사칭`
- `기타`

### 3차 라벨 (Technique)
- `접근`: 신뢰 획득 시도
- `협박`: 공포 조성
- `개인정보요구`: PII 요청
- `송금유도`: 금전 요구

### 4차 라벨 (Severity)
- `LOW`: 의심 수준
- `MEDIUM`: 주의 필요
- `HIGH`: 명백한 피싱
- `CRITICAL`: 즉각 대응 필요

3. 📝 텍스트 기반 피싱 사례 수집 (100건)

### Phase 2: 단기 (다음 주)

5. 📝 YouTube 공개 자료 수집
6. 📝 자체 제작 데이터 확장 (50건)

### Phase 3: 중기 (2주 내)
7. 📝 데이터 증강 적용

- `scripts/collect_fss_data.py` - 금융감독원 데이터
- `scripts/collect_aihub_data.py` - AI Hub 데이터

- `scripts/augment_data.py` - 데이터 증강

## 법적 고려사항

### 저작권
- AI Hub: 공공 데이터, 이용약관 준수
- 금융감독원: 공익 목적, 출처 명시
- YouTube: 저작권 확인, 교육/연구 목적 명시
- 자체 제작: 문제 없음

### 개인정보보호
- 모든 개인정보는 마스킹 처리
- 실제 전화번호, 계좌번호 등 제거
- 음성 변조 (필요시)
- 익명화 처리

## 데이터 품질 관리

### 검증 기준
1. 음성 품질: 8kHz 이상
2. 명료도: 70% 이상 단어 인식률
3. 라벨 정확도: 2명 이상 교차 검증
4. 중복 제거: 유사도 95% 이상 제거

### 주기적 업데이트
- 월 1회 새로운 피싱 사례 추가
- 분기 1회 전체 데이터 재검증
- 년 1회 라벨링 체계 개선
